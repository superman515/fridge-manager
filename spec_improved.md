# 냉장고 관리 페이지 MVP Spec (개선)

## 📋 개요
2인 가족의 주부를 위한 냉장고 내 음식 관리 웹 페이지. 유통기한 관리 및 중복 구매 방지에 초점.

---

## 🎯 문제 정의
- **문제 1**: 음식 유통기한을 자주 까먹음
- **문제 2**: 냉장고에 뭐가 있는지 모르면 중복 구매 발생

---

## 👤 타겟 사용자
**페르소나**: 2인 가족 (주부 + 가족 구성원)
- 주 1-2회 장을 봄
- 냉장고 관리에 신경 쓰고 싶으나 유통기한 확인이 번거로움
- 가족 모두가 냉장고 상태를 실시간으로 공유하고 싶음

---

## ✨ 핵심 기능 (MVP)

### 1. 음식 추가 / 삭제
- 음식명 입력
- 카테고리 선택 (채소, 육류, 유제품, 기타)
- 유통기한 날짜 입력
- 음식 삭제 기능

### 2. 유통기한 표시
- 남은 일수 계산 및 표시
- **유통기한 경과 음식**: 빨간색 강조 표시
- **임박한 음식** (3일 이내): 주황색 표시 (권장)
- **안전한 음식**: 일반 표시

### 3. 카테고리 필터
- 필터 드롭다운으로 카테고리 선택 (채소, 육류, 유제품, 기타, 전체)
- 선택한 카테고리의 음식만 표시

### 4. 정렬 기준 선택
- 유통기한 임박순 (가장 임박한 순서)
- 추가 순서 (최신 추가순)
- 카테고리별 (채소 → 육류 → 유제품 → 기타)

### 5. 사용자 계정 & 로그인
- Google/이메일 로그인 (Firebase Authentication)
- 자동 로그인 (세션 유지)

### 6. 가족 공유 기능
- 가족 그룹 생성 및 초대 코드 발급
- 가족 구성원 초대 (이메일 기반)
- 모든 가족 구성원이 같은 냉장고 데이터 실시간 공유
- 구성원 목록 보기 및 제거 기능

---

## 🚫 제외 범위 (이번 MVP에서는 안 함)
- 통계 / 분석 (버린 음식 통계 등)
- 바코드 스캔 기능
- 모바일 알림
- 권한 관리 (모든 멤버가 동일한 권한)

---

## ✅ 성공 기준 (2주 뒤 시연)
- [ ] 가족 구성원 2명 이상이 각자 계정으로 로그인한다
- [ ] 한 명이 음식을 추가하면 다른 구성원이 실시간으로 같은 음식을 본다
- [ ] 냉장고에 음식을 5개 추가하고 카테고리별로 보인다
- [ ] 유통기한이 지난 음식에 빨간색으로 표시된다
- [ ] 삭제한 음식이 모든 구성원의 화면에서 사라진다

## 🛠️ 기술 스택

| 항목 | 기술 |
|------|------|
| 프론트엔드 | Next.js (App Router) + TypeScript |
| 상태 관리 | React useState / useEffect |
| 인증 | Firebase Authentication (Google/이메일 로그인) |
| 데이터 저장 | Firebase Firestore (실시간 동기화) |
| 스타일링 | CSS / Tailwind CSS |

### 아키텍처
- 모든 냉장고 데이터는 Firebase Firestore에 저장
- 가족 그룹 기반 데이터 공유 (그룹 ID로 필터링)
- 실시간 리스너로 다른 멤버의 변경사항 즉시 동기화
- Vercel에 배포 (Next.js 호스팅)

## 🎬 사용 흐름 (Happy Path)

### 첫 사용자 (가족 대표)
1. 페이지 접속 → 로그인/회원가입 (Google/이메일)
2. "가족 그룹 생성" 버튼 클릭
3. 그룹명 입력 후 생성 → 초대 코드 받음
4. "음식 추가" 버튼 클릭
5. 음식명, 카테고리, 유통기한 입력 후 저장
6. 필터 & 정렬로 목록 관리

### 초대된 가족 구성원
1. 페이지 접속 → 로그인/회원가입
2. "초대 코드 입력" 또는 "가족 링크 클릭"
3. 즉시 대표가 추가한 음식 목록 공유
4. 실시간으로 다른 멤버의 변경사항 반영
5. 음식 추가/삭제 시 모든 멤버에게 즉시 반영
