# 냉장고 관리 페이지 MVP Spec

## 📋 개요
2인 가족의 주부를 위한 냉장고 내 음식 관리 웹 페이지. 유통기한 관리 및 중복 구매 방지에 초점.

---

## 🎯 문제 정의
- **문제 1**: 음식 유통기한을 자주 까먹음
- **문제 2**: 냉장고에 뭐가 있는지 모르면 중복 구매 발생

---

## 👤 타겟 사용자
**페르소나**: 2인 가족의 주부
- 주 1-2회 장을 봄
- 냉장고 관리에 신경 쓰고 싶으나 유통기한 확인이 번거로움
- 나중에 가족과 공유 기능을 추가할 가능성 있음

---

## ✨ 핵심 기능 (MVP)

### 1. 음식 추가 / 삭제
- 음식명 입력
- 카테고리 선택 (채소, 육류, 유제품, 기타)
- 유통기한 날짜 입력
- 음식 삭제 기능

### 2. 유통기한 표시
- 남은 일수 계산 및 표시
- **유통기한 경과 음식**: 빨간색 강조 표시
- **임박한 음식** (3일 이내): 주황색 표시 (권장)
- **안전한 음식**: 일반 표시

### 3. 카테고리 분류
- 카테고리별 탭 또는 필터 (채소, 육류, 유제품, 기타)
- 전체 보기 옵션

---

## 🚫 제외 범위 (이번 MVP에서는 안 함)
- 사용자 계정 & 로그인 기능
- 통계 / 분석 (버린 음식 통계 등)
- 바코드 스캔 기능
- 가족 공유 기능 (나중에 추가 가능)
- 모바일 알림

---

## ✅ 성공 기준 (3일 뒤 시연)
- [ ] 냉장고에 음식을 5개 추가하고 카테고리별로 보인다
- [ ] 유통기한이 지난 음식에 빨간색으로 표시된다
- [ ] 삭제한 음식이 목록에서 사라진다

---

## 🛠️ 기술 스택

| 항목 | 기술 |
|------|------|
| 프론트엔드 | Next.js (App Router) + TypeScript |
| 상태 관리 | React useState / useEffect |
| 데이터 저장 | 브라우저 로컬 스토리지 |
| 스타일링 | CSS / Tailwind CSS (기존 프로젝트 스타일 유지) |

### 향후 확장
- 서버/API 필요 시점: 가족 공유 기능 추가 시
- 기존 Next.js 구조에서 `/api` 라우트 추가로 쉽게 확장 가능

---

## 📅 개발 일정
**기간**: 3일
- Day 1: UI 구성 + 음식 추가/삭제 기능
- Day 2: 유통기한 계산 + 색상 표시 로직
- Day 3: 카테고리 필터 + 테스트 & 버그 수정

---

## 🎬 사용 흐름 (Happy Path)
1. 페이지 접속
2. "음식 추가" 버튼 클릭
3. 음식명, 카테고리, 유통기한 입력 후 저장
4. 목록에서 카테고리별로 음식 확인
5. 유통기한 경과 음식 확인 및 삭제
6. 브라우저 닫았다 다시 열어도 데이터 유지 (로컬 스토리지)
