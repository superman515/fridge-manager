# 냉장고 관리 페이지 MVP - PRD (Product Requirements Document)

## Core Narrative (핵심 네러티브)

2인 가족의 주부는 냉장고에 뭐가 있는지, 언제까지 먹을 수 있는지 정확히 알지 못해 중복 구매와 식품 낭비를 반복한다. 더 근본적인 문제는 **가족 구성원 간에 냉장고 정보가 실시간으로 공유되지 않는다**는 것이다. 메모지나 카톡으로 "우유 3개 있음(48일)"이라 전달해도, 숫자를 기억하고 비교하는 인지적 부담이 크다. 이 앱은 **'시각적 상태 표시(색상)'를 핵심**으로, 유통기한을 색상(안전/임박/경과)으로 순간적으로 판단할 수 있게 만든다. 한 명이 음식을 추가하면 가족 모두의 화면에 즉시 반영되고, 색상으로 우선순위를 판단한다. 그 결과 **월 5~10만원의 불필요한 식품 낭비를 방지**하고, 가족이 협력해서 냉장고를 관리할 수 있게 된다.

### 핵심 요소

- **대상**: 바쁜 일상 속에서 냉장고 관리를 소홀히 하기 쉬운, 가족 식사 관리를 책임지는 주부
- **근본 문제**: 가족 간 냉장고 정보 동기화 부재로 인한 중복 구매·식품 낭비와 이를 인지적 부담 없이 공유하지 못함
- **핵심 관점**: 메모나 숫자 전달보다 **시각적 상태 표시(색상)**가 정보 처리의 인지적 부담을 획기적으로 줄임
- **해결 방향**: 중앙화된 목록 + 실시간 공유 + 색상 기반 상태 표시로 가족 모두가 자동으로 최신 냉장고 정보를 공유하고 빠르게 판단할 수 있게 함
- **성공의 모습**: 가족이 협력해서 냉장고를 관리하고, 월 5~10만원의 불필요한 지출을 줄임으로써 가계 관리가 한 층 수월해짐

## Problem Statement

2인 가족의 주부가 다음과 같은 문제를 직면하고 있습니다:

1. **유통기한 관리의 어려움**: 냉장고에 들어간 음식의 유통기한을 자주 까먹어 식품낭비 발생
2. **가족 간 정보 불일치**: 냉장고에 뭐가 있는지 몰라서 중복 구매 발생하고, 가족 간에 냉장고 상태를 공유할 방법이 없음
3. **수동 관리의 번거로움**: 유통기한 확인이 번거로워서 냉장고 관리를 포기하게 됨

현재 해결 방법이 없어, 음식 낭비와 불필요한 지출이 계속됨.

## Solution

**냉장고 관리 웹 페이지**를 통해 다음을 제공합니다:

1. **중앙화된 음식 목록**: 냉장고에 있는 모든 음식을 한곳에서 관리
2. **시각적 유통기한 관리**: 유통기한까지의 남은 일수를 색상으로 표시 (안전/임박/경과)
3. **가족 공유 기능**: 모든 가족 구성원이 실시간으로 같은 냉장고 데이터 공유
4. **간편한 필터 & 정렬**: 카테고리별 필터링 및 유통기한순 정렬로 빠른 확인

Firebase를 통한 실시간 동기화로, 한 명이 음식을 추가하면 다른 가족 구성원들이 즉시 확인할 수 있습니다.

## User Stories

1. As a 주부, I want to 음식을 추가할 때 음식명, 카테고리, 유통기한을 입력할 수 있기를 원합니다, so that 냉장고에 들어간 음식을 체계적으로 기록할 수 있습니다

2. As a 주부, I want to 각 음식의 유통기한까지 남은 일수를 한눈에 볼 수 있기를 원합니다, so that 어떤 음식을 먼저 먹어야 할지 빠르게 판단할 수 있습니다

3. As a 주부, I want to 유통기한이 경과한 음식이 빨간색으로 표시되기를 원합니다, so that 위험한 음식을 즉시 식별할 수 있습니다

4. As a 주부, I want to 유통기한이 3일 이내로 남은 음식이 주황색으로 표시되기를 원합니다, so that 우선적으로 먹어야 할 음식을 강조할 수 있습니다

5. As a 주부, I want to 카테고리 필터를 사용해서 특정 종류의 음식만 볼 수 있기를 원합니다 (채소, 육류, 유제품, 기타), so that 필요한 음식만 빠르게 찾을 수 있습니다

6. As a 주부, I want to 유통기한 임박순, 추가 순서, 카테고리별로 목록을 정렬할 수 있기를 원합니다, so that 음식을 효율적으로 관리할 수 있습니다

7. As a 주부, I want to 음식을 삭제할 수 있기를 원합니다, so that 먹은 음식이나 버린 음식을 목록에서 제거할 수 있습니다

8. As a 사용자, I want to 구글 로그인 또는 이메일 로그인으로 계정을 만들 수 있기를 원합니다, so that 안전하게 내 계정을 관리할 수 있습니다

9. As a 사용자, I want to 로그인 상태를 유지할 수 있기를 원합니다, so that 매번 로그인할 필요가 없습니다

10. As a 가족 대표, I want to 가족 그룹을 생성할 수 있기를 원합니다, so that 가족 구성원들과 냉장고 데이터를 공유할 수 있습니다

11. As a 가족 대표, I want to 가족 그룹을 생성할 때 초대 코드를 받을 수 있기를 원합니다, so that 다른 가족 구성원들을 쉽게 초대할 수 있습니다

12. As a 가족 구성원, I want to 초대 코드를 입력해서 가족 그룹에 참여할 수 있기를 원합니다, so that 가족의 냉장고 정보에 접근할 수 있습니다

13. As a 가족 구성원, I want to 내가 추가한 음식이 다른 가족 구성원의 화면에 실시간으로 나타나기를 원합니다, so that 가족 모두가 항상 최신 냉장고 정보를 공유할 수 있습니다

14. As a 가족 대표, I want to 가족 그룹의 구성원 목록을 볼 수 있기를 원합니다, so that 누가 내 냉장고 정보에 접근하고 있는지 알 수 있습니다

15. As a 가족 대표, I want to 가족 그룹에서 구성원을 제거할 수 있기를 원합니다, so that 필요할 때 접근 권한을 제거할 수 있습니다

16. As a 가족 구성원, I want to 다른 구성원이 삭제한 음식이 내 화면에서도 실시간으로 사라지기를 원합니다, so that 모든 구성원이 항상 같은 냉장고 상태를 본다는 확신을 가질 수 있습니다

17. As a 사용자, I want to 반응형 UI에서 편하게 이용할 수 있기를 원합니다, so that 데스크탑과 태블릿에서 모두 쉽게 관리할 수 있습니다

18. As a 가족 대표, I want to 가족 그룹을 수정 또는 삭제할 수 있기를 원합니다, so that 그룹 설정을 관리할 수 있습니다

19. As a 사용자, I want to 내 프로필 정보를 보고 수정할 수 있기를 원합니다, so that 계정 정보를 관리할 수 있습니다

20. As a 사용자, I want to 로그아웃할 수 있기를 원합니다, so that 공유 기기에서 안전하게 로그아웃할 수 있습니다

## Implementation Decisions

### 1. 기술 스택

- **프론트엔드**: Next.js (App Router) + TypeScript
- **상태 관리**: React useState / useEffect (외부 라이브러리 미사용)
- **인증**: Firebase Authentication (Google/이메일 제공자)
- **데이터 저장**: Firebase Firestore (실시간 동기화)
- **스타일링**: CSS 또는 Tailwind CSS
- **배포**: Vercel

### 2. 아키텍처 설계

#### 데이터 모델

**사용자 (User)**
- uid (Firebase UID)
- email
- displayName
- photoURL
- familyGroupId (속해있는 가족 그룹의 ID, 초기값: null)

**가족 그룹 (FamilyGroup)**
- id (자동 생성)
- name (그룹명)
- createdBy (생성자의 uid)
- members (구성원 uid 배열)
- inviteCode (초대 코드)
- createdAt

**음식 (Food)**
- id (자동 생성)
- familyGroupId (속한 가족 그룹)
- name (음식명)
- category (채소/육류/유제품/기타)
- expiryDate (유통기한 날짜, ISO 8601 형식)
- addedBy (추가한 사용자의 uid)
- addedAt (추가 날짜)

#### 보안 규칙

- **사용자**: 자신의 데이터만 읽기/쓰기 가능
- **가족 그룹**: 멤버에만 읽기 가능, 생성자만 쓰기 가능
- **음식**: familyGroupId로 필터링하여 해당 그룹 멤버만 읽기/쓰기 가능

### 3. 페이지 구조

- `/` — 랜딩/로그인 페이지
- `/auth/login` — 로그인 페이지
- `/auth/signup` — 회원가입 페이지
- `/app/dashboard` — 냉장고 관리 메인 페이지 (보호된 페이지)
- `/app/family` — 가족 그룹 관리 페이지 (보호된 페이지)
- `/app/profile` — 프로필 관리 페이지 (보호된 페이지)

### 4. UI 컴포넌트 구조

**주요 컴포넌트**:
- `FoodForm` — 음식 추가 폼
- `FoodCard` — 개별 음식 카드
- `FoodList` — 음식 목록
- `Filter` — 카테고리 필터
- `SortOptions` — 정렬 옵션
- `FamilyGroupCreator` — 가족 그룹 생성 모달
- `FamilyGroupInvite` — 가족 그룹 초대 모달
- `FamilyMemberList` — 가족 구성원 목록

### 5. 실시간 동기화

- Firebase Firestore의 `onSnapshot` 리스너를 사용하여 데이터 변경을 실시간으로 감지
- 각 가족 그룹의 음식 데이터에 리스너 등록
- 변경 발생 시 즉시 UI 업데이트

### 6. 유통기한 계산 및 색상 표시

```typescript
type FoodStatus = 'expired' | 'urgent' | 'safe';

function getFoodStatus(expiryDate: Date): FoodStatus {
  const today = new Date();
  const daysLeft = Math.ceil((expiryDate.getTime() - today.getTime()) / (1000 * 60 * 60 * 24));
  
  if (daysLeft < 0) return 'expired';
  if (daysLeft <= 3) return 'urgent';
  return 'safe';
}

// 색상 매핑
// expired (빨간색 #EF4444)
// urgent (주황색 #F97316)
// safe (초록색 #22C55E)
```

### 7. 초대 코드 생성

- 6자리 또는 8자리 고유한 코드 생성 (대문자 영문 + 숫자)
- 수동 입력 또는 QR 코드 스캔 지원 (옵션)

### 8. 에러 처리

- Firebase 인증 에러: 명확한 에러 메시지 표시
- Firestore 연결 실패: 재시도 또는 오프라인 모드 표시
- 사용자 입력 검증: 클라이언트 사이드 검증

#### 필터 & 정렬
- 카테고리 필터 적용

- 구성원 제거
- 실시간 데이터 동기화 (다중 클라이언트 시뮬레이션)

#### 통합 테스트
- 2명 이상의 사용자가 동시에 로그인 → 음식 추가/삭제 → 실시간 반영 확인
- 로그아웃 후 재로그인 시 이전 데이터 유지

### 3. 테스트 환경

- Firebase 에뮬레이터 사용 (로컬 테스트 환경)
- 또는 Firebase 프로젝트의 테스트 환경 (staging)

### 4. E2E 테스트

- Playwright 또는 Cypress로 전체 흐름 테스트
- 시나리오: 회원가입 → 그룹 생성 → 초대 → 음식 추가 → 다른 사용자로 확인

## Out of Scope

다음 기능들은 이번 MVP에서 제외됩니다:

1. **통계 & 분석**: 버린 음식 통계, 지출 분석, 쇼핑 제안
2. **바코드 스캔**: 음식 정보 자동 입력
3. **모바일 푸시 알림**: 유통기한 임박 알림
4. **권한 관리**: 모든 멤버는 동일한 권한 (데이터 추가/삭제 가능)
5. **다국어 지원**: 한국어만 지원
6. **오프라인 모드**: 인터넷 연결 필수
7. **AI 기반 기능**: 음식 자동 인식, 추천
8. **이미지 업로드**: 음식 사진 저장
9. **구독 기능**: 무료로만 제공
10. **소셜 기능**: 친구 추가, 냉장고 공유 요청
